A trigger determines when a flow runs.


You specify the service account that runs the flow, and which conditions make the flow run.


For example, you can specify flows to be triggered when you mention a service account
in a discussion, or when you assign the service account as a reviewer.


Create a trigger


{{< history >}}


    

  • Assign and Assign reviewer event types introduced in GitLab 18.5.
    • 



{{< /history >}}


Prerequisites:


    

  • You must have at least the Maintainer role for the project.
    • 



To create a trigger:


    

  1. On the top bar, select Search or go to and find your project.
    2. 

  2. Select Automate > Triggers.
    3. 

  3. Select New flow trigger.
    4. 

  4. In Description, enter a description for the trigger.
    5. 

  5. From the Event types dropdown list, select one or more event types:
    6. 

  6. Mention: When the service account user is mentioned
     in a comment on an issue or merge request.
    7. 

  7. Assign: When the service account user is assigned
     to an issue or merge request.
    8. 

  8. Assign reviewer: When the service account user is assigned
     as a reviewer to a merge request.
    9. 

  9. From the Service account user dropdown list,
   select the service account user.
    10. 

  10. For Configuration source, select one of the following:
    11. 

  11. AI Catalog: From the flows configured for this project,
     select a flow for the trigger to execute.
    12. 

  12. Configuration path: Enter the path to the flow configuration file
     (for example, .gitlab/duo/flows/claude.yaml).
    13. 

  13. Select Create flow trigger.
    14. 



The trigger now appears in Automate > Triggers.


Edit a trigger


    

  1. On the top bar, select Search or go to and find your project.
    2. 

  2. Select Automate > Triggers.
    3. 

  3. For the trigger you want to change, select Edit flow trigger ({{< icon name="pencil" >}}).
    4. 

  4. Make the changes and select Save changes.
    5. 



Delete a trigger


    

  1. On the top bar, select Search or go to and find your project.
    2. 

  2. Select Automate > Triggers.
    3. 

  3. For the trigger you want to change, select Delete flow trigger ({{< icon name="remove" >}}).
    4. 

  4. On the confirmation dialog, select OK.
    5.
