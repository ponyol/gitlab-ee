Manage languages for Code Suggestions


{{< history >}}


    

  • Introduced in GitLab Workflow for VS Code 4.21.0
    • 



{{< /history >}}


You can customize your coding experience in VS Code by enabling or disabling Code Suggestions for specific supported languages.
You can do this by editing your settings.json file directly, or from the VS Code user interface:


    

  1. In VS Code, open the extension settings for GitLab Workflow:
    2. 

  2. On the top bar, go to Code > Settings > Extensions.
    3. 

  3. Search for GitLab Workflow in the list, and select Manage ({{< icon name="settings" >}}).
    4. 

  4. Select Extension Settings.
    5. 

  5. In your User settings, find the section titled AI Assisted Code Suggestions: Enabled Supported Languages.
    6. 

  6. To enable Code Suggestions for a language, select its checkbox.
    7. 

  7. To disable Code Suggestions for a language, clear its checkbox.
    8. 

  8. Your changes are automatically saved, and take effect immediately.
    9. 



When you disable Code Suggestions for a language, the GitLab Duo icon changes to show that suggestions are disabled
for this language. On hover, it shows Code Suggestions are disabled for this language.


Add support for more languages


If your desired language doesn't have Code Suggestions available by default,
you can add support for your language locally.
However, Code Suggestions might not function as expected.

To do this:


    

  1. Find your desired language in the list of
   language identifiers.
   You need the Identifier for your languages in a later step.
    2. 

  2. In VS Code, open the extension settings for GitLab Workflow:
    3. 

  3. On the top bar, go to Code > Settings > Extensions.
    4. 

  4. Search for GitLab Workflow in the list, and select Manage ({{< icon name="settings" >}}).
    5. 

  5. Select Extension Settings.
    6. 

  6. In your User settings, find
      GitLab â€º Ai Assisted Code Suggestions: Additional Languages and select Add Item.
    7. 

  7. In Item, add the identifier for each language you want to support. Identifiers should be
   lowercase, like html or powershell. Don't add leading periods from file suffixes to each identifier.
    8. 

  8. Select OK.
    9.

{{< tab title="JetBrains IDEs" >}}


Prerequisites:


    

  • You have installed and enabled the
  GitLab plugin for JetBrains IDEs.
    • 

  • You have completed the Jetbrains extension setup
  instructions, and authorized the extension to access your GitLab account.
    • 



To do this:


    

  1. Find your desired language in the list of
   language identifiers.
   You need the identifier for your languages in a later step.
    2. 

  2. In your IDE, on the top bar, select your IDE name, then select Settings.
    3. 

  3. On the left sidebar, select Tools > GitLab Duo.
    4. 

  4. Under Code Suggestions Enabled Languages > Additional languages, add the identifier for each language
   you want to support. Identifiers should be in lowercase, like html. Separate multiple identifiers with commas,
   like html,powershell,latex, and don't add leading periods to each identifier.
    5. 

  5. Select OK.
    6. 



{{< /tab >}}


{{< tab title="Eclipse" >}}


Prerequisites:


    

  • You have installed and enabled the GitLab for Eclipse plugin.
    • 

  • You have completed the Eclipse setup
  instructions, and authorized the extension to access your GitLab account.
    • 



To do this:


    

  1. In your Eclipse bottom menu, select the GitLab icon.
    2. 

  2. Select Show Settings.
    3. 

  3. Scroll down to the Code Suggestions Enabled Languages section.
    4. 

  4. In Additional Languages, add a comma-separated list of language identifiers. Don't
   add leading periods to the identifiers. For example, use html, md, and powershell.
    5.
