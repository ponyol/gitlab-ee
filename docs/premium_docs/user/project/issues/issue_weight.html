When you have a lot of issues, it can be hard to get an overview.
With weighted issues, you can get a better idea of how much time,
value, or complexity a given issue has or costs. You can also sort by weight
to see which issues need to be prioritized.


View the issue weight


You can view the issue weight on:


    

  • The right sidebar of each issue.
    • 

  • The issues page, next to a weight icon ({{< icon name="weight" >}}).
    • 

  • Issue boards, next to a weight icon ({{< icon name="weight" >}}).
    • 

  • The milestone page, as a total sum of issue weights.
    • 



Set the issue weight


{{< history >}}


    

  • Minimum role to set issue weight changed from Reporter to Planner in GitLab 17.7.
    • 



{{< /history >}}


Prerequisites:


    

  • You must have at least the Planner role for the project.
    • 



You can set the issue weight when you create or edit an issue.


You must enter whole, positive numbers.


When you change the weight of an issue, the new value overwrites the previous value.


When you create an issue


To set the issue weight when you create an issue, enter a
number under Weight.


From an existing issue


To set the issue weight from an existing issue:


    

  1. Go to the issue.
    2. 

  2. On the right sidebar, in the Weight section, select Edit.
    3. 

  3. Enter the new weight.
    4. 

  4. Select any area outside the dropdown list.
    5. 



From an issue board


To set the issue weight when you edit an issue from an issue board:


    

  1. Go to your issue board.
    2. 

  2. Select an issue card (not its title).
    3. 

  3. On the right sidebar, in the Weight section, select Edit.
    4. 

  4. Enter the new weight.
    5. 

  5. Select any area outside the dropdown list.
    6.
