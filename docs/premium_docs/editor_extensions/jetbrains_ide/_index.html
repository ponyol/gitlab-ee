title: GitLab Duo plugin for JetBrains IDEs




The GitLab Duo plugin integrates GitLab Duo
with JetBrains IDEs like IntelliJ, PyCharm, GoLand, Webstorm, and Rubymine.


To install and configure the extension, see Install and set up.


Use with remote development


The GitLab Duo plugin works with JetBrains Remote Development when installed on the host machine (remote server).


For information about installing plugins in remote development environments, see the JetBrains documentation:


    

  • Install plugins in remote projects.
    • 

  • Add plugins to Dev Containers.
    • 



Enable experimental or beta features


Some features in the plugin are in experiment or beta status. To use them, you must opt in:


    

  1. Go to your IDE's top menu bar and select Settings, or:
    2. 

  2. MacOS: press Command+,
    3. 

  3. Windows or Linux: press Control+Alt+S
    4. 

  4. On the left sidebar, expand Tools, then select GitLab Duo.
    5. 

  5. Select Enable Experiment or BETA features.
    6. 

  6. To apply the changes, restart your IDE.
    7. 



Update the extension


To update your extension to the latest version:


    

  1. In your JetBrains IDE, go to Settings > Plugins.
    2. 

  2. From Marketplace, select GitLab Duo published by GitLab, Inc..
    3. 

  3. Select Update to update to the latest plugin version.
    4. 



Enable telemetry


The GitLab Duo plugin uses the telemetry settings in your JetBrains IDE to send usage and error
information to GitLab. To enable telemetry in your JetBrains IDE:


    

  1. Go to your IDE's top menu bar and select Settings. For example, in PyCharm, select PyCharm > Settings.
    2. 

  2. On the left sidebar, expand Tools, then select GitLab Duo.
    3. 

  3. Under Advanced, select the Enable telemetry checkbox.
    4. 

  4. Select OK or Apply to save your changes.
    5.

You can configure the plugin to use 1Password secret references for authentication instead of
hard-coding personal access tokens.


Prerequisites:


    

  • You have the 1Password desktop app installed.
    • 

  • You have the 1Password CLI tool installed.
    • 



To integrate GitLab for JetBrains with the 1Password CLI:


    

  1. Authenticate with GitLab. Either:
    2. 

  2. Install the glab CLI and
     configure the 1Password shell plugin.
    3. 

  3. Follow the GitLab for JetBrains set up steps.
    4. 

  4. Open the 1Password item.
    5. 

  5. Copy the secret reference.
    6. 



If you use the gitlab 1Password shell plugin, the token is stored as a password under "op://Private/GitLab Personal Access Token/token".


From the IDE:


    

  1. Go to your IDE's top menu bar and select Settings.
    2. 

  2. On the left sidebar, expand Tools, then select GitLab Duo.
    3. 

  3. Under Authentication, select the 1Password CLI tab.
    4. 

  4. Select Integrate with 1Password CLI.
    5. 

  5. Optional. For Secret reference, paste the secret reference you copied from 1Password.
    6. 

  6. Optional. To verify your credentials, select Verify setup.
    7. 

  7. Select OK or Save.
    8. 



Report issues with the plugin


You can report any issues, bugs, or feature requests in the
gitlab-jetbrains-plugin issue tracker.
Use the Bug or Feature Proposal template.


If you encounter an error while using GitLab Duo, you can also report it with your IDE's
built-in error reporting tool:


    

  1. To access the tool, either:
    2. 

  2. When an error occurs, in the error message, select See details and submit report.
    3. 

  3. In the status bar, on the bottom right, select the exclamation mark.
    4. 

  4. In the IDE Internal Errors dialog, describe the error.
    5. 

  5. Select Report and clear all.
    6. 

  6. Your browser opens a GitLab issue form, pre-populated with debug information.
    7. 

  7. Follow the prompts in the issue template to fill out the description, providing
   as much context as you can.
    8. 

  8. Select Create issue to file the bug report.
    9.
